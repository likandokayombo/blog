# Understanding React Server Components

React Server Components (RSC) are a **new paradigm** in React that allows components to run on the server and be streamed to the client. They are designed to improve performance by reducing JavaScript sent to the browser.

---

## Why Use React Server Components?

- **Reduced bundle size**: Only client-side code is sent to the browser.
- **Better performance**: Heavy computations happen on the server.
- **Simpler data fetching**: Fetch data directly in components without worrying about client-side requests.

> “Server Components let you offload work to the server while keeping the UI interactive.” – React Docs

## How to Create a Server Component

A Server Component is just a React component saved with the `.server.jsx` or `.server.tsx` extension. For example:

```tsx
// app/components/CurrentTime.server.tsx
export default function CurrentTime() {
  const now = new Date().toLocaleTimeString();
  return <p>Current server time: {now}</p>;
}

// app/page.tsx
import CurrentTime from './components/CurrentTime.server';

export default function Page() {
  return (
    <div>
      <h2>Welcome to React Server Components</h2>
      <CurrentTime />
    </div>
  );
}
```
